## 🎯 Problema Focado

A principal dor do torcedor abordada neste projeto é a **perda de momentos importantes da partida**, causada pelo deslocamento dentro do estádio e pelo tempo gasto em filas para compra de alimentos.

Além disso, também foram considerados:

* Dificuldade de localização no ambiente do estádio
* Sobrecarga de informação em momentos de alta emoção

---

## 🧠 Justificativa de Design

A organização das informações foi baseada em princípios de usabilidade voltados para **contextos de alta distração e pressão**, como um estádio durante uma partida.

As decisões de design incluem:

* **Hierarquia visual clara**, com destaque para o placar e tempo de jogo
* **Acesso rápido às funções principais** por meio de atalhos na tela inicial
* **Elementos visuais simples (ícones e botões grandes)** para facilitar a interação
* **Redução de carga cognitiva**, evitando excesso de texto
* **Fluxos curtos e diretos**, minimizando o número de etapas necessárias

Essas escolhas garantem que o usuário consiga interagir com o aplicativo de forma eficiente, mesmo em um ambiente com ruído, movimento e distrações constantes.

## 🔄 Fluxo do Usuário

### 🍔 Solicitação de alimentos:

1. O usuário acessa a tela **Home/Dashboard**
2. Seleciona a opção **“Pedir Comida”**
3. Navega pelo **Cardápio Digital**
4. Adiciona os itens desejados ao carrinho
5. Acessa a tela de **Carrinho/Pagamento**
6. Escolhe o método de entrega (retirada ou no assento)
7. Finaliza o pedido
8. Recebe uma notificação quando o pedido estiver pronto

---

### 🎥 Visualização de replay:

1. O usuário acessa a tela **Home/Dashboard**
2. Seleciona a opção **“Replays”**
3. Visualiza a lista de lances disponíveis
4. Escolhe o replay desejado
5. Assiste ao conteúdo em tempo real
